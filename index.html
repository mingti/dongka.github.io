问题描述

![[source/attachments/Pasted image 20251231111953.png]]

从一系列连续区间中，检索出与指定区间 $A$ 有重叠的部分。

示例：

区间组：1-5, 6-20, 21-40, 41-58

目标 $A$：15-30

结果：6-20, 21-40 （因其与 $A$ 存在范围重合）

使用 start 和 stop 就可以表示一个区间,但我这里增加了gap, 可以把gap当做当前行的一个”数值”, 例如数量或金额, 这样stop就可以看做是累计的数量或金额.

同时采用另外的写法将A写入了#con临时表.

方法1

这是一种朴素的方法. a.stop >= 任意子区间的start, a.start <=任意子区间的 stop.

图像可以表示, A必须”卡住”t1 ,才能符合要求, 如果没有”卡住”, 那么表示不重合.
![[26094818-37a8-4bf8-b04c-a040209b3dd0.gif]]

同理我们可以得知, 以下两种不重合的情况需要排除.

  1. t1完全在A的左边
    ![[Pasted image 20251230220423.png]]

  2. t1完全在A的右边
    ![[Pasted image 20251230215642.png]]

这时我们观察a.stop-t1.stop, 它的大小必须介于 -t1.gap和 a.gap 之间, 为什么是-t1.gap, 因为在t1完全在A的右边的情况, a.stop-t1.stop相减为负数.

这种写法只用到了gap和stop , 即当前的数值和累计数值, 更贴合实际.

例题:

将发票金额依次分配给收款单

on的两种写法对应上面的两种方法.

1 . val 看作是gap, 累计看作stop , 则start=累计-val

IIF来计算每次分配的金额. 这里有两种写法,

  1. 第一种的思路是当a.累计发票金额 >= b.累计收款金额, 说明目前的钱够分配, 应分配的金额为累计收款金额 - 上一次的结算点, 通过累计发票金额 - a.val得到上一次的累计发票金额,即上一次的结算点

当条件false时, 即a.累计发票金额 < b.累计收款金额, 说明钱不够分配, 那应该能分配多少就分配多少, 这里还要分两种情况, 一个是上一次分配完还留了点, 加上这次自己全部的钱也不够,这种情况分配这次全部的钱, 即a.val 这里最后是通过 a.累计发票金额 - (累计发票金额 - a.val) 得到的a.val

![[Pasted image 20251230225927.png]]

还有一种情况是上一次没有分配够, 这次需要补上, 分配的钱就是累计发票金额 - 上一次的累计收款金额

![[Pasted image 20251230230007.png]]

这两种子情况本质上是用累计发票金额 - MAX( 上一次的累计收款金额, 上一次的累计发票金额, ) SQL server 2019 没有求两个值的MAX函数, 所以用了IFF替代.

  1. 这个的思路是每次应分配的钱是两个区间的重合部分. 比较简单, MIN(这次) - MAX(上一次)

递归CTE执行顺序

递归CTE的执行步骤如下：

  1. 初始化：执行递归CTE中的非递归部分（即UNION ALL前面的部分），生成初始结果集（也称为“锚点”部分）。

  2. 递归：将上一步的结果集作为输入，执行递归部分（即UNION ALL后面的部分），生成新的结果集。

  3. 重复步骤2，直到递归部分返回的结果集为空。

  4. 合并：将所有步骤中生成的结果集合并（UNION ALL）为最终结果。
